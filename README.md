# GML CSV 點位畫布工具

本工具用於 GML 整理前處理：將廠商提供的 GML 實測點位 CSV，
轉換成可人工判讀的互動式 HTML 畫布，並自動產生異常清單與重複量測比對表，
供人工確認與後續整理使用。

> ⚠️ 本工具是「判讀輔助工具」，不是「最終審查工具」。
> 程式只會標示異常與彙整清單，**不會**自動刪除重複點、不會自動取平均座標、
> 也不會自行判斷哪一筆資料才是正確的。所有判斷都保留給人工確認。

## 2. 放置 CSV 檔案

請將廠商提供的所有 CSV 檔案，放到專案的：

```
input_csv/
```

資料夾內。可以放多個 CSV，程式會自動讀取此資料夾內所有 `.csv` 檔案。

- **不會修改任何原始 CSV**，所有讀取皆為唯讀。
- CSV 編碼會自動嘗試 `utf-8-sig`、`utf-8`、`big5`、`cp950`、`gb18030`，
  通常常見的 Excel 存檔（Big5 / UTF-8）皆可正常讀取。

### 欄位名稱

程式會自動偵測下列欄位（含常見別名）：

| 標準欄位 | 常見對應名稱（部分範例） |
|---|---|
| 識別碼 | 識別碼、ID、點號、點名、編號 |
| 類別碼 | 類別碼、類別、CODE |
| X | X、E、X(E)、X（E） |
| Y | Y、N、Y(N)、Y（N） |
| Z | Z、H、Z(H)、Z（H）、高程 |
| 日期 | 日期、測量日期、設置日期、施測日期 |

若你的 CSV 欄位名稱不在上述清單中，**不需要修改程式**，
只要打開 `config/rules.yaml`，在對應欄位的清單中加入該 CSV 實際使用的欄位名稱即可。

## 4. 輸出結果

程式執行完成後，會在 `output/` 資料夾產生以下檔案：

| 檔案 | 說明 |
|---|---|
| `點位畫布.html` | 互動式 Plotly 畫布。可縮放、平移；同一管線識別碼的點位依點號排序並以線段連接；滑鼠移到點位上會顯示該點的詳細資訊（原始識別碼、管線識別碼、點號、類別碼、X/Y/Z、測量日期、來源 CSV、原始列號）。直接用瀏覽器開啟即可。 |
| `解析後點位表.xlsx` | 所有 CSV 解析後的完整點位資料，包含管線識別碼、點號、座標（轉換後數值與原始字串）、日期、來源 CSV 與原始列號。 |
| `異常清單.xlsx` | 各項異常檢查結果，包含異常類型、說明、相關點位資訊與建議處理方式。 |
| `重複量測比對表.xlsx` | 同一「管線識別碼＋點號」出現多筆資料時的兩兩比對結果，包含 XY 平面差距、Z 高程差距與判定結果（近似重複 / 差異過大），所有重複資料皆保留，不會自動刪除或平均。 |

## 5. 識別碼解析規則

識別碼解析規則：**抓取結尾的連續數字作為「點號」，其餘部分為「管線識別碼」**。

範例：

```
1-9-30mTE04~TB06-C1   ->  管線識別碼: 1-9-30mTE04~TB06-C   點號: 1
1-9-30mTE04~TB06-C6   ->  管線識別碼: 1-9-30mTE04~TB06-C   點號: 6
```

若識別碼結尾沒有數字，則整串視為管線識別碼、點號留空，
並會在異常清單中以「欄位缺漏」標示，提醒人工確認該識別碼格式。

---

## 6. 異常檢查項目

程式會檢查並列在 `異常清單.xlsx` 中：

- 欄位缺漏（CSV 中找不到對應欄位）
- 座標空值（X 或 Y 為空）
- 日期空值
- X / Y 座標格式異常（非數字）
- 同一管線識別碼內，各點位日期不一致
- 同一管線識別碼＋點號重複
- 重複點位 XY 平面差距與 Z 高程差距比對結果：
  - XY ≤ 0.30m 且 Z ≤ 0.10m → 「近似重複，待確認」
  - XY > 0.30m 或 Z > 0.10m → 「重複點位座標差異過大，待確認」

所有重複點位的明細，會完整列在 `重複量測比對表.xlsx` 中，供人工逐一確認。

## 7. 設定檔說明（config/rules.yaml）

`config/rules.yaml` 內容包含：

- `field_aliases`：各標準欄位（識別碼、類別碼、X、Y、Z、日期）可接受的欄位名稱清單，
  可自行新增廠商 CSV 實際使用的欄位名稱。
- `id_parse.trailing_digit_regex`：識別碼解析用的正規表示式，預設為抓取結尾連續數字。
- `duplicate_thresholds`：重複點位判定門檻（`xy_tolerance_m`、`z_tolerance_m`）。
- `csv_encodings`：讀取 CSV 時依序嘗試的編碼。

修改設定檔後，重新執行 `python src/main.py` 即可套用新設定。

## 8. 常見問題（FAQ）

**Q1. 執行後顯示「找不到對應欄位」警告怎麼辦？**

代表程式在某個 CSV 中找不到對應的標準欄位（例如 X、Y、Z 或日期）。
請打開該 CSV 確認實際欄位名稱，然後到 `config/rules.yaml` 對應欄位下方新增該名稱，
重新執行即可。同時該檔案所有點位也會在「異常清單」中以「欄位缺漏」標示。

**Q2. CSV 是用 Excel 存的，會不會讀不到？**

不會。程式會依序嘗試 `utf-8-sig`、`utf-8`、`big5`、`cp950`、`gb18030` 等編碼，
大部分 Excel 在 Windows 上存的 CSV（Big5 或 UTF-8）都可以正常讀取。
若所有編碼皆失敗，程式仍會嘗試以容錯方式讀入，並在終端機顯示警告，
建議檢查該 CSV 的編碼或內容是否異常。

**Q3. 為什麼有些點位沒有畫在 HTML 畫布上？**

座標為空值或格式異常（無法轉換為數字）的點位不會畫在畫布上，
但這些點位仍會出現在「異常清單.xlsx」中，請依異常清單逐一確認原始 CSV。

**Q4. 發現重複點位，程式會自動處理嗎？**

不會。程式只會將重複的「管線識別碼＋點號」列在「重複量測比對表.xlsx」中，
並標示判定結果（近似重複 / 差異過大），但保留哪一筆、是否需要補測，
都需要人工確認後再行處理。

**Q5. 我可以重複執行程式嗎？**

可以。每次執行都會重新讀取 `input_csv/` 內所有 CSV，並重新產生 `output/` 內的所有檔案
（會覆蓋舊的輸出檔案），不會影響原始 CSV。

**Q6. 識別碼結尾沒有數字（無法解析出點號）怎麼辦？**

該筆資料的「管線識別碼」會直接使用整個識別碼字串，「點號」留空，
並會在異常清單中以「欄位缺漏」標示，提醒人工確認該識別碼格式是否正確。

## 9. 專案結構

```
gml-csv-canvas-tool/
├─ input_csv/              # 放置廠商提供的 CSV（不會被修改）
├─ output/                 # 程式輸出結果
├─ config/
│  └─ rules.yaml           # 欄位對應、解析規則、重複點位門檻等設定
├─ src/
│  ├─ main.py              # 主程式進入點
│  ├─ csv_loader.py        # 讀取 CSV、欄位自動偵測
│  ├─ id_parser.py         # 識別碼解析（管線識別碼 / 點號）
│  ├─ anomaly_detector.py  # 異常檢查與重複點位比對
│  ├─ canvas_generator.py  # Plotly 互動式畫布產生
│  └─ excel_exporter.py    # Excel 輸出（openpyxl）
├─ requirements.txt
└─ README.md
```

## 10. 第一版不包含的功能

- 不建立資料庫、不建立登入系統、不做多人權限、不做線上平台
- 不支援拖拉修改點位
- 不直接輸出正式 GML 格式
- 不處理量測照片
- 不修改原始 CSV
- 不自動刪除重複點位、不自動取平均座標、不自行判斷哪一筆重複資料為正確版本

## 11. 第二版功能說明與操作流程

第二版在第一版基礎上，**新增兩份 Excel 範本**，供人工判讀與後續修正使用。
程式仍不會自動修改任何原始 CSV。

### 11.1 新增輸出檔案

| 檔案 | 說明 |
|---|---|
| `output/人工判讀紀錄範本.xlsx` | 以所有解析後點位為底，逐列進行人工判讀，填寫判讀狀態與問題類型。 |
| `output/修正紀錄範本.xlsx` | 空白範本，人工記錄每一筆修正動作（新增 / 刪除 / 採用 / 排除 / 修正）。 |

### 11.2 人工判讀紀錄範本欄位說明

| 欄位 | 說明 |
|---|---|
| 來源CSV | 程式自動帶入（資料來源欄，灰底） |
| 原始列號 | 程式自動帶入 |
| 原始識別碼 | 程式自動帶入 |
| 管線識別碼 | 程式自動帶入 |
| 點號 | 程式自動帶入 |
| X / Y / Z | 程式自動帶入座標數值 |
| 測量日期 | 程式自動帶入 |
| **判讀狀態** | **人工填寫（黃底）**，含下拉選單：正常 / 待確認 / 需廠商補測 / 排除 / 採用 |
| **問題類型** | **人工填寫（黃底）**，含下拉選單：點位偏移 / 點位缺漏 / 點位重複 / 座標異常 / 日期不一致 / 點序錯誤 / 其他 |
| **判讀說明** | **人工填寫**，自由描述該筆問題細節 |
| **建議回覆施工廠商內容** | **人工填寫**，供後續回覆廠商時使用 |
| **確認人員** | **人工填寫**，填入判讀負責人姓名 |
| **確認日期** | **人工填寫**，填入判讀完成日期 |

### 11.3 修正紀錄範本欄位說明

| 欄位 | 說明 |
|---|---|
| **動作類型** | 含下拉選單：新增 / 刪除 / 採用 / 排除 / 修正 |
| 原始識別碼 | 對應來源資料識別碼 |
| 管線識別碼 | 對應管線識別碼 |
| 點號 | 對應點號 |
| 原X / 原Y / 原Z | 原始座標（可從解析後點位表複製） |
| 修正後X / 修正後Y / 修正後Z | 修正後座標（若動作為刪除 / 排除則留空） |
| 修正原因 | 說明修正依據或判讀依據 |
| 是否納入後續GML | 填寫「是」或「否」，供後續 GML 整理人員使用 |
| 確認人員 | 填入修正負責人姓名 |
| 確認日期 | 填入修正完成日期 |

### 11.4 第二版操作流程

```
Step 1  放置 CSV → 執行程式
        python src/main.py
        ↓ 產生：
        output/點位畫布.html          ← 第一版，互動式畫布
        output/解析後點位表.xlsx      ← 第一版，所有點位資料
        output/異常清單.xlsx          ← 第一版，自動異常偵測
        output/重複量測比對表.xlsx    ← 第一版，重複點位比對
        output/人工判讀紀錄範本.xlsx  ← 第二版，人工判讀用
        output/修正紀錄範本.xlsx      ← 第二版，修正紀錄用

Step 2  開啟「點位畫布.html」
        在瀏覽器中逐管線逐點位目視確認，
        搭配「異常清單.xlsx」確認自動偵測到的問題。

Step 3  填寫「人工判讀紀錄範本.xlsx」
        - 對每筆點位選擇「判讀狀態」（黃底下拉選單）
        - 有問題的點位選擇「問題類型」並填寫「判讀說明」
        - 填寫「建議回覆施工廠商內容」供日後追蹤
        - 確認後填入「確認人員」與「確認日期」
        ⚠️ 不要修改灰底欄位（來源CSV / 原始列號 / 原始識別碼等）

Step 4  填寫「修正紀錄範本.xlsx」
        - 每一筆修正動作新增一列
        - 選擇「動作類型」（新增 / 刪除 / 採用 / 排除 / 修正）
        - 填入原始座標與修正後座標（若有）
        - 填寫「修正原因」與「是否納入後續GML」
        ⚠️ 本範本是人工登記表，程式不會讀取此檔案自動修改任何資料

Step 5  依據修正紀錄，人工整理最終點位表
        後續 GML 整理人員依「修正紀錄範本.xlsx」
        逐筆手動處理最終版本，確保資料品質有據可查。
```

> ⚠️ 重要提醒：
> - 程式不會讀取「人工判讀紀錄範本」或「修正紀錄範本」中的內容。
> - 這兩份範本是「人工作業表單」，供判讀人員記錄與追蹤使用。
> - 所有修改決策皆由人工確認後手動執行，程式永遠不修改原始 CSV。
