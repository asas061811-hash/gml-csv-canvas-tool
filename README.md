# GML 點位處理工具 V0.6 版

廠商 GML 實測點位 CSV 前處理工具。  
將 CSV 點位資料視覺化為互動式 HTML 畫布，並自動產生異常清單與重複量測比對表。

> **本工具目前為內部測試版，尚未正式發布。**  
> 目前功能分為兩大頁籤：  
> 1. **資料整理** — CSV 點位檢核、人工判讀、修正、分類表與 Excel 下載。  
> 2. **GML 輸出** — GML 專用寬表、規範選擇、GML 預覽與 GML 下載。

> **重要提醒**：本工具為**判讀輔助工具**，所有異常標示僅供參考。  
> 重複量測採用哪一筆、座標是否可接受、是否需請廠商補測，**一律由人工確認**。

## 一、安裝 Python 套件

### 前提：已安裝 Python 3.8 以上版本

開啟命令提示字元（cmd）或 PowerShell，切換到本工具目錄：

```
cd gml-csv-canvas-tool
```

安裝所需套件：

```
pip install -r requirements.txt
```

安裝的套件：
| 套件 | 用途 |
|------|------|
| pandas | 讀取與處理 CSV |
| plotly | 產生互動式 HTML 畫布 |
| openpyxl | 輸出 Excel 檔案 |
| PyYAML | 讀取設定檔 |

---

## 二、放置 CSV 檔案

將廠商提供的 CSV 量測檔案放入：

```
gml-csv-canvas-tool/
└─ input_csv/        ← 把所有 CSV 放這裡
      廠商量測01.csv
      廠商量測02.csv
      ...
```

- 支援多個 CSV 同時處理
- 支援 UTF-8、UTF-8-SIG（BOM）、Big5、CP950 等編碼
- **不會修改原始 CSV**

## 三、執行程式

### 方法 A：Streamlit 網頁版（建議）

雙擊 `start_web.bat`，然後在瀏覽器開啟 `http://localhost:8501`

或命令列：

```
cd gml-csv-canvas-tool
python -m streamlit run app.py
```

網頁版功能：
- 直接在瀏覽器上傳 CSV，不需手動複製檔案
- 互動式點位畫布（縮放、平移、hover）
- 人工判讀表格可在瀏覽器直接填寫下拉選單
- 修正紀錄可新增列、直接輸入
- 一鍵下載全部成果 Excel 檔案

### 方法 B：命令列版（CLI）

雙擊 `run.bat`，或命令列：

```
cd gml-csv-canvas-tool
python src/main.py
```

CLI 版會直接讀取 `input_csv/` 資料夾，成果輸出至 `output/`。

執行過程會顯示：
- 各 CSV 的讀取狀態與欄位對應
- 偵測到的異常數量
- 輸出檔案位置

## 四、成果檔案

執行完成後，`output/` 資料夾內會產生：

```
output/
├─ 點位畫布.html           ← 互動式點位瀏覽畫布
├─ 解析後點位表.xlsx       ← 所有點位的解析與整理結果
├─ 異常清單.xlsx           ← 需人工確認的異常清單
├─ 重複量測比對表.xlsx     ← 重複點的座標比對
├─ 人工判讀紀錄範本.xlsx   ← [v2] 逐點判讀紀錄，含下拉選單
└─ 修正紀錄範本.xlsx       ← [v2] 修正動作紀錄，含下拉選單
```

### 點位畫布.html
- 用瀏覽器（Chrome / Edge）開啟
- 支援縮放、平移
- 點選/滑鼠懸停顯示點位詳細資訊
- 圖例可點選顯示/隱藏個別管線
- 紅色 ✕ 符號 = 有異常的點位
- 可另存為 PNG 圖片（右上角工具列）

### 解析後點位表.xlsx
所有點位整理後的資料，欄位包含：
原始識別碼、管線識別碼、點號、類別碼、X(E)、Y(N)、Z(H)、測量日期、來源CSV、原始列號

### 異常清單.xlsx
偵測到的異常，欄位包含：
異常類型、異常說明、原始識別碼、管線識別碼、點號、X、Y、Z、測量日期、來源CSV、原始列號、建議處理方式

偵測的異常類型：
| 異常類型 | 說明 |
|---------|------|
| 欄位缺漏 | CSV 找不到對應的標準欄位 |
| 識別碼空值 | 識別碼欄位為空 |
| 座標空值 | X、Y 或 Z 欄位為空 |
| X/Y 座標格式異常 | 座標值無法解析為數字 |
| 日期空值 | 測量日期欄位為空 |
| 同一管線日期不一致 | 同一管線存在多個測量日期 |
| 同一管線識別碼 + 點號重複 | 相同管線+點號出現多筆資料 |
| 重複點位座標差異過大 | 重複點的座標差距超過容差 |

### 人工判讀紀錄範本.xlsx（v2 新增）

預填所有點位資料，供人工逐點填寫判讀結果：
- **淡藍色欄位**：請人工填寫（判讀狀態、問題類型、說明等）
- **淡黃色列**：程式偵測到有異常，已預填「待確認」，請優先確認
- **白色列**：程式未偵測到異常，可依目視判讀填寫

欄位說明：

| 欄位 | 類型 | 說明 |
|------|------|------|
| 來源CSV～測量日期 | 唯讀 | 程式自動帶入，請勿修改 |
| 判讀狀態 | **下拉選單** | 正常 / 待確認 / 需廠商補測 / 排除 / 採用 |
| 問題類型 | **下拉選單** | 點位偏移 / 點位缺漏 / 點位重複 / 座標異常 / 日期不一致 / 點序錯誤 / 其他 |
| 判讀說明 | 文字 | 自由填寫說明 |
| 建議回覆施工廠商內容 | 文字 | 供撰寫回覆廠商的參考文字 |
| 確認人員 | 文字 | 填寫確認人員姓名 |
| 確認日期 | 文字 | 填寫確認日期（建議格式：YYYY-MM-DD）|

### 修正紀錄範本.xlsx（v2 新增）

空白範本，供人工填寫每一個修正決策（每列一筆動作）：

| 欄位 | 類型 | 說明 |
|------|------|------|
| 動作類型 | **下拉選單** | 新增 / 刪除 / 採用 / 排除 / 修正 |
| 原始識別碼～點號 | 文字 | 填寫對應的點位識別資訊 |
| 原X / 原Y / 原Z | 數字 | 填寫修正前的座標 |
| 修正後X / 修正後Y / 修正後Z | 數字 | 填寫修正後的座標（若無修改可留空）|
| 修正原因 | 文字 | 說明修正原因 |
| 是否納入後續GML | **下拉選單** | 是 / 否 |
| 確認人員 | 文字 | 填寫確認人員姓名 |
| 確認日期 | 文字 | 填寫確認日期 |

> **注意**：兩份範本均不會修改原始 CSV，僅作為人工確認的紀錄文件。

### 重複量測比對表.xlsx
所有重複點的詳細比對，欄位包含：
來源CSV、原始列號、原始識別碼、管線識別碼、點號、測量日期、X、Y、Z、比對基準、XY差距(m)、Z差距(m)、判定結果、建議處理方式

判定結果說明：
- **近似重複，待確認**：XY 差距 ≤ 0.30m 且 Z 差距 ≤ 0.10m
- **重複點位座標差異過大，待確認**：XY 差距 > 0.30m 或 Z 差距 > 0.10m

## 五、欄位對應設定

若程式無法自動識別欄位名稱，可修改設定檔：

```
config/rules.yaml
```

在對應欄位下新增 CSV 中使用的欄位名稱：

```yaml
column_aliases:
  x:
    - X
    - E
    - X(E)
    - 東坐標      ← 新增你的欄位名稱
  y:
    - Y
    - N
    - Y(N)
    - 北坐標      ← 新增你的欄位名稱
```

也可調整重複點的距離容差：

```yaml
thresholds:
  xy_tolerance: 0.30   # 平面距離容差（公尺）
  z_tolerance: 0.10    # 高程容差（公尺）
```

## 六、識別碼解析規則

程式以**末尾連續數字為點號，其餘為管線識別碼**：

| 原始識別碼 | 管線識別碼 | 點號 |
|-----------|-----------|------|
| `1-9-30mTE04~TB06-C1` | `1-9-30mTE04~TB06-C` | `1` |
| `1-9-30mTE04~TB06-C6` | `1-9-30mTE04~TB06-C` | `6` |
| `1-9-30mTE04~TB06-C12` | `1-9-30mTE04~TB06-C` | `12` |

若識別碼末尾無數字，點號欄位會空白，該點仍會顯示在畫布上（但無法連線）。

## 七、常見問題

**Q：執行時出現「找不到 python 命令」**  
A：請確認已安裝 Python，可在命令列輸入 `python --version` 確認。

**Q：執行時出現「ModuleNotFoundError」**  
A：請先執行 `pip install -r requirements.txt` 安裝套件。

**Q：CSV 讀取後欄位全是空白**  
A：可能是編碼問題。程式會自動嘗試 UTF-8、Big5 等編碼，若仍失敗，請用 Excel 另存為 UTF-8 CSV 後再試。

**Q：點位畫布顯示的點位位置看起來不對**  
A：請確認 X 對應東（E）座標、Y 對應北（N）座標。若座標系不同，可在 `config/rules.yaml` 調整欄位對應。

**Q：想調整重複點的判定距離**  
A：修改 `config/rules.yaml` 的 `thresholds.xy_tolerance` 和 `thresholds.z_tolerance`。

**Q：output/ 資料夾找不到輸出檔案**  
A：請確認執行時無錯誤訊息，且 `input_csv/` 內有 CSV 檔案。

## 九、v2 操作流程

### 完整判讀流程（建議順序）

```
執行程式
   ↓
【1. 點位畫布.html】
   → 用瀏覽器開啟
   → 目視確認點位分佈是否合理
   → 確認連線走向是否正確
   → 紅色 ✕ 符號標示的異常點先重點確認

【2. 異常清單.xlsx】
   → 確認每筆異常的性質
   → 判斷是否需要廠商補測

【3. 重複量測比對表.xlsx】
   → 逐組確認重複量測的原因
   → 決定採用哪一筆（不修改 CSV，紀錄於修正範本）

【4. 人工判讀紀錄範本.xlsx】（v2 新增）
   → 逐點填寫「判讀狀態」下拉選單
   → 若有問題，選擇「問題類型」並填寫說明
   → 填寫「建議回覆施工廠商內容」（若需聯絡廠商）
   → 填寫「確認人員」與「確認日期」

【5. 修正紀錄範本.xlsx】（v2 新增）
   → 依判讀結果，每筆修正填寫一列
   → 選擇「動作類型」（新增/刪除/採用/排除/修正）
   → 填寫座標修正前後值
   → 標注是否納入後續 GML
   → 填寫「確認人員」與「確認日期」
```

### 判讀狀態選用建議

| 狀態 | 使用時機 |
|------|----------|
| **正常** | 座標合理，可直接採用 |
| **待確認** | 有疑問，尚未決定 |
| **需廠商補測** | 資料缺漏或明顯異常，需廠商重新測量 |
| **排除** | 確認此點不採用（如重複點選用其中一筆後，其餘排除）|
| **採用** | 確認採用（如重複點選定採用的那筆）|

### 修正動作類型說明

| 動作類型 | 使用時機 |
|----------|----------|
| **新增** | 補充一個原本缺少的點位 |
| **刪除** | 此點確認不納入（重複點的棄用筆）|
| **採用** | 明確標記此點採用（重複點的採用筆）|
| **排除** | 此點暫時排除，不納入 GML |
| **修正** | 座標有誤，填寫修正後的正確座標 |


## 十一、分類表欄位規則

1. 正式分類表不顯示 UI 操作欄、point_key、內部追蹤欄位。
2. 管線分類表前 37 欄為屬性欄位（不足以「擴充欄位N」填滿）。
3. 第 38 欄（Excel AM 欄）開始排列點位座標。
4. 座標欄位依各點位順序重複排列為：E、N、Z、d、E、N、Z、d、E、N、Z、d……
5. 點狀設施（人手孔、開關閥等）只有正式屬性欄位 + X/Y/Z/d，不含來源CSV等內部欄位。

## 十二、設施屬性合併與三版本成果

### 正式流程

```
工具完成點位整理 → 產生分類表點位資料
        ↓
下載分類表，人工填寫屬性（填定義文字，如「使用」、「地下」、「mm」）
        ↓
上傳「分類表_定義版」回工具
        ↓
工具自動合併屬性 + 代碼轉換
        ↓
產生三版本：定義版 / 代碼版 / 代碼定義對照版
        ↓
GML 輸出使用「代碼版」
```

### 三版本說明

| 版本 | 用途 | 欄位內容 |
|------|------|---------|
| 定義版 | BIM、人工檢查 | 使用狀態=使用、管線型態=地下 |
| 代碼版 | 廠商確認、GML 輸出 | 使用狀態=0、管線型態=0 |
| 代碼定義對照版 | 內部查核 | 使用狀態=0、使用狀態_定義=使用 |

### 注意事項

1. 人工建議填定義內容，不建議優先填代碼。
2. 工具會自動把定義內容轉成代碼。
3. GML 輸出使用代碼版。
4. 若定義內容無法轉代碼，會列入檢核訊息。
5. 代碼對照規則定義在 `config/rules.yaml`。

---

## 十三、點位排序轉向

1. 可針對同一條管線執行點位排序轉向（在右側編輯面板中操作）。
2. 例如 1→2→3→4→5→6 可轉成 6→5→4→3→2→1。
3. 系統會重新指派點號，使轉向後仍從 1 開始。
4. 轉向後會同步更新畫布、分類表、Excel、HTML 與 GML 輸出。
5. 原始 CSV 不會被修改。
6. 所有轉向操作都會寫入修正紀錄。

## 十四、GML 產製

### GML 產製頁籤

在 Streamlit 網頁版的「🌐 GML 產製」頁籤，可直接從修正後資料產生 GML 檔案。

**目前支援：**
- 目標規範：桃園市（全部 9 種設施類型）
- 國土署、南科/世曦規範已建立介面，尚未完整實作

**功能：**

| 功能 | 說明 |
|------|------|
| 資料來源 | 使用目前修正後資料，或上傳 GML 專用寬表 CSV |
| 目標規範 | 桃園市 / 國土署 / 南科·世曦 |
| Z 值高程模式 | 絕對高程（直接輸出 Z）或地面高程（輸出 Z − d） |
| GML 預覽 | 產製後顯示 XML 預覽 |
| GML 下載 | 一鍵下載 `.gml` 檔案 |
| GML 專用寬表 CSV | 下載管線寬表（點位橫向展開）或點位資料 CSV |

### GML 專用寬表 CSV

管線資料以一條管線一列的方式輸出，點位座標橫向展開：

```
類別碼, 識別碼, ..., 第1點X, 第1點Y, 第1點Z, 第1點d, 第2點X, 第2點Y, ...
```

### Z 值高程模式

| 模式 | 說明 |
|------|------|
| 絕對高程 | Z 值已是絕對高程，GML 直接使用 Z |
| 地面高程 | Z 為地面高程，GML 輸出 Z − d（埋設深度）|

桃園市規範固定輸出四維座標 `X Y Z d`，不受高程模式影響。

### 重要規則

- **已排除資料不會進入 GML 產製**
- **原始 CSV 不會被修改**
- GML 產製使用修正後的資料（包含人工編輯結果）

## 十五、人工新增點位

工具支援在不修改原始 CSV 的情況下，人工新增點位至任一管線。

### 適用情境

- 原始 CSV 缺少連接點，需補端點或轉折點
- 需與其他管線、人孔、節點或設施銜接
- 廠商資料不完整，需手動補充測量點

### 操作方式

上傳 CSV 並完成處理後，切換右側面板至「**➕ 新增點位**」頁籤。

**方式 A：複製目前點位，新增下一點**

1. 在畫布點選某一點位（例如第 6 點）
2. 切換至「➕ 新增點位」頁籤
3. 填入新點的 X / Y / Z 座標
4. 點擊「新增至管線尾端（點號 7）」
5. 確認後，第 7 點自動加入

**方式 B：空白表單新增**

1. 切換至「➕ 新增點位」頁籤
2. 選擇目標管線識別碼（或手動輸入）
3. 填寫類別碼、座標、測量日期等
4. 確認後新增

**插入中間點位**

1. 點選某點位（例如第 3 點）
2. 展開「🔀 插入點位（在第 3 點之後）」
3. 填入座標後確認插入
4. 後續點位（第 4, 5, 6, …）自動重新編號為第 5, 6, 7, …

### 規則說明

| 項目 | 規則 |
|------|------|
| point_key | 格式 `manual_{timestamp}_{8位hex}`，新增後永不改變 |
| 來源 CSV | 標記為「人工新增」，不修改原始 CSV |
| 原始列號 | 標記為「人工新增」 |
| 預設點號 | 同一管線目前最大點號 + 1 |
| 識別碼 | 留空時自動產生：`{管線識別碼}.{點號}` |
| 判讀狀態 | 自動設為「已修正」 |
| 問題類型 | 自動設為「人工新增點位」 |
| 是否納入成果 | 預設「是」 |

### 同步更新範圍

人工新增點位（`是否納入成果 = 是`）自動進入：

- 修正後點位表
- 點位畫布（綠色菱形 ◆）
- 分類表（管線橫向展開座標）
- GML 專用寬表 CSV
- GML 輸出 LineString 座標
- Excel 下載成果
- 修正紀錄（含插入時的重新編號紀錄）

### 清除

點擊「🗑️ 清除全部修正」後，所有人工新增點位也會一併移除。
